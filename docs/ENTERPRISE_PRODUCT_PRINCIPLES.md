# Enterprise Product Principles

Version

---

# 我們打造的是什麼？

PetPulse Enterprise OS

是一套

Enterprise Decision Operating System

而不是：

- Dashboard
- 報表系統
- BI 工具
- Social Listening Tool

資料不是目的。

決策才是目的。

---

# 核心使命

協助主管：

用最少的認知負擔，

完成更好的企業決策。

因此，

所有設計都必須服務於：

Decision Making。

---

# Product Philosophy

我們相信：

企業主管不是缺資料。

而是缺：

清楚的判斷。

因此，

PetPulse 永遠遵守：

Information

↓

Insight

↓

Decision

↓

Action

而不是：

Data

↓

Data

↓

More Data

---

# Executive First

每一頁都先回答：

今天發生什麼？

需要決策什麼？

下一步去哪裡？

而不是：

今天收集多少資料。

---

# Information Architecture

所有 Workspace

固定：

Header

↓

Executive Summary

↓

KPI

↓

Main Content

↓

Next Action

不得任意變更。

原因：

主管閱讀應形成固定節奏。

---

# Less Cognitive Load

任何畫面：

都應減少：

閱讀成本

切換成本

理解成本

思考成本

避免：

資訊爆炸。

---

# One Decision Per Screen

每個區塊，

只回答一件事情。

例如：

企業健康

↓

今天健康嗎？

Evidence

↓

為什麼？

Workspace

↓

去哪裡？

不要混在一起。

---

# Streamlit Native First

優先：

Streamlit 原生元件。

避免：

大量 HTML

大量 CSS

大量 JavaScript

原因：

可維護。

一致。

容易交接。

---

# Theme First

品牌：

由 Theme 控制。

Workspace

不得自行指定品牌色。

避免：

每頁長得不一樣。

---

# Copy First

每一句文字，

都應該：

幫助主管做決策。

而不是：

描述系統。

例如：

不寫：

Evidence Count

改寫：

今日證據數量

不寫：

High Risk

改寫：

高風險事件

---

# Decision Before Analysis

先：

今天要不要處理？

再：

為什麼？

最後：

完整分析。

---

# Product Consistency

任何新 Workspace

都必須：

看起來像：

PetPulse。

不是：

另一套產品。
